# Go rules

Derived from [go-repo-conventions.md](../../docs/research/spikes/go-repo-conventions.md),
which cites the projects each convention came from. Read it when a rule here
seems arbitrary — the reasoning is there.

## 1. The design laws are structural, not aspirational

L1–L8 in the PRD are enforced by package boundaries and linters wherever that
is possible. A law that only exists in a comment is a law that will be broken
by someone in a hurry, including you.

**`internal/temporal` and `internal/acl` are pure.** They contain functions over
domain types. They import no database driver, no `database/sql`, no `pgx`, and
take no connection in any signature. `depguard` fails the build if that changes.

**The store returns rows, never decisions.** There is no
`GetVisibleClaims(principal)`. There is `LoadClaims(ids)` and
`LoadEvidence(claimIDs)`, and `internal/acl` computes
`claim.acl ⊆ ⋂(evidence_i.acl)` in Go.

**L5 is never a SQL predicate.** This costs performance — rows cross the wire
that Postgres could have discarded. It is affordable at one instance per
organization, and the alternative is the known silent-failure path: pgvector
filtering under ACL selectivity returns 4 results where 40 were expected, with
no error. Deciding in SQL is not merely harder to test.

Row-level security is a **backstop that must never be the only check**. The
Go-side intersection is the decision; RLS catches programmer error. The RLS
coverage test asserts a redundancy, and its comment must say so.

**The principal type lives in the engine.** D-014's standing check, run against
CRED itself and not only competitors: *grep the engine for the principal type;
if it only appears in a client package, the retreat has happened.*

## 2. Layout

```
main.go                  thin: flags, calls internal/cli
internal/
  cli/                   subcommands: serve, curate, doctor, migrate
  config/                env + flag resolution, CRED_* prefix
  claim/                 Claim, Evidence, Derivation — domain types
  temporal/              bi-temporal algebra. PURE.
  acl/                   ACL intersection algebra. PURE.
  recall/                retrieval orchestration, RRF fusion
  nominate/              the LLM boundary: Nominate() + fake + adapters
  curate/                River workers
  store/
    migrations/          *.sql, embedded via embed.FS
    pg/                  the ONLY package importing pgx
  embed/                 onnx-gomlx path + tokenizer
  mcpsrv/                MCP tools, transport, auth
  obs/                   slog + OTel setup, attribute constants
testdata/
```

- **`main.go` at the root, not `cmd/cred/main.go`.** One binary with two
  subcommands (`serve`, `curate`), not two binaries. Add `cmd/` the day a second
  binary genuinely exists; the move is one `git mv`.
- **No `pkg/`.** The official Go layout guidance never mentions it, and
  `golang-standards/project-layout` disclaims its own authority. `internal/` is
  compiler-enforced; `pkg/` is decoration.
- **No `api/`, `build/`, `scripts/`, `test/`, `third_party/`.**
- **`obs/` holds every OTel attribute name as a constant.** The `gen_ai.*`
  conventions have no stable release and already renamed `gen_ai.system` to
  `gen_ai.provider.name`. One rename should touch one file.

## 3. CGO is off, and the guard needs two assertions

`CGO_ENABLED=0` everywhere. It buys cross-compilation without QEMU, distroless
static bases, and musl compatibility — losing it loses all three at once.

Two checks that look correct and are not:

- `go tool nm | grep cgo` reports `_cgo_` symbols with CGO **on and off**.
- `go list -f '{{if .CgoFiles}}...' -deps ./...` run under `CGO_ENABLED=0`
  writes to **stderr** and emits nothing on stdout when a cgo-only dependency is
  present, so a `[ -z "$(... 2>/dev/null)" ]` test passes while the build is
  broken.

The guard is **two assertions**: enumerate cgo packages under `CGO_ENABLED=1`,
then assert `CGO_ENABLED=0 go build` succeeds.

## 4. Embeddings and the tokenizer

The read path requires **no API key, no provider choice, and no vector-store
decision**. This is the strongest unoccupied position in the research and it is
lost the first time someone adds a remote provider "just for now".

**Tokenizer character-class tables are generated by probing the pinned reference
tokenizer.** Never `unicode.Is`. A textbook-correct implementation scored
99.67%, and every failure was invisible on English text — the worst failure
shape there is. The generator is checked in; its output is regenerated, never
hand-edited.

**int8 quantization is a memory lever, not a latency lever.** It is ~2x slower
in `simplego` because `onnx-gomlx` widens both operands to int32 before
multiplying.

## 5. Errors

Wrap with `%w` when the caller can act on the cause; do not wrap when it merely
decorates. Sentinel errors for conditions callers branch on, typed errors when
they need the detail. `errors.Is` for identity, `errors.As` for extraction —
never string matching.

Do not add `pkg/errors`. It is dormant, with no functional release since
January 2020.

## 6. Testing

Follow [testing-strategy.md](../../docs/research/spikes/testing-strategy.md)
for suite shape and the bi-temporal invariants. Budget is 2–5 minutes per
commit.

- **Table-driven by default**, standard library first.
- **`testify/require` is allowed**, under `testifylint`. Five of seven surveyed
  `go.mod` files depend on it, including pgx, River, and goose. The "serious
  projects avoid testify" claim traces to the Google style guide, which says do
  not *create* assertion libraries and never names testify. Do not use
  `testify/assert` — a failed assertion that continues produces cascading noise.
- **A skipped test and a passing test are the same green check.** Integration
  tests may skip locally when Postgres is absent; in CI, `CRED_REQUIRE_DB=1`
  turns a skip into a failure.
- **The temporal and ACL algebra is unit-tested as pure functions.** If a test
  for either needs a database, the boundary has already been violated.

## 7. Database

**pgx directly**, not sqlc, and not an ORM. `sqlc` maps `vector` but has no
`halfvec` or `sparsevec` support, and `tech-decisions.md` commits CRED to
`halfvec`. Revisit if that changes.

`goose` for migrations, embedded via `embed.FS` so the single binary carries
them. `squawk` reviews migrations for locking hazards.

Pin `pgvector/pgvector:0.8.5-pg17`. PG18's PGDATA change causes silent data
loss.

`embedding_model_id` is NOT NULL on every vector row. No defaults.

## 8. Context, logging, observability

`context.Context` is the first parameter, never stored in a struct. Whoever
starts a goroutine owns its lifetime and its shutdown path.

`log/slog` only. Structured fields, no formatted strings carrying data.

**Never log claim or evidence content.** Log identifiers, counts, and
durations. Ingested content is untrusted (L8) and logs are read by tools that
were not designed to treat it as such.

## 9. Dependencies

Every dependency is a maintenance liability for a solo maintainer. The accepted
set is the standard library, `pgx`, `goose`, `River`, the MCP Go SDK, the
`onnx-gomlx` embedding stack, and `testify/require`.

Anything beyond that needs a reason recorded in the commit message. Prefer
copying thirty lines over adding a module for them.

## 10. Working here

**Write code that reads like the code around it.** Match the surrounding comment
density and naming.

**Comment the *why*, not the *what*.** The non-obvious reasons in this codebase
are almost always a design law or a measured failure — say which. `// L5: the
intersection is computed here, never in SQL` is worth more than three lines
describing the loop.

**When a rule here blocks something that seems clearly better, say so rather
than working around it.** Several of these rules exist because a prior approach
failed measurably, and several others are judgement that has not been tested
yet. The decision log records which is which.
